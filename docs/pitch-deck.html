SmartFlow AI

A data-driven AI operations assistant for SMEs and producer cooperatives — customer communication, orders, shipments, stock, and daily tasks in a single workflow.

Pitch deck · 10 slides · July 2026 · Live demo: smartflow-frontend-production.up.railway.app

Problem

Small businesses typically run their operations across disconnected tools and manual tracking. This weakens the customer experience, stock visibility, and management response time.

Delayed responses Messages, order status, and shipment status live in different places, so customer questions can't be answered quickly or consistently.
Late-detected risk Critical stock shortages and shipment delays usually surface only after a customer complains.
Scattered tasks Daily operational priorities depend on the person and the day, with no consolidated briefing.
Lost productivity Searching for, copying, and reporting the same data over and over drains the team's capacity.

Solution: SmartFlow AI

A single AI agent that understands the customer's message, queries the business's real operational data (orders, products, shipments, stock) through structured tool calls, and produces action-oriented output for both the customer and the manager.

Core value proposition

  • For customers: verifiable, data-backed answers (estimated delivery, delay days, stock availability).
  • For managers: a summary dashboard, critical stock and shipment alerts, supplier email drafts, and a daily operations briefing.
  • Not a scripted chatbot: a flow wired directly to the data layer via function calling.

Target audience & usage

Who is it for?

  • Small and mid-sized producer / seller businesses.
  • Producer cooperatives with a shared warehouse–shipping model.
  • Teams handling high daily volume across the order + shipment + stock triangle.

User roles

  • Customer / end user: natural-language questions about orders, products, and shipments.
  • Operations / manager: dashboard, task list, alerts, and daily briefing.

MVP focus: a stable, deployed demo — single agent, structured data on SQLite; scaling and further integrations are the next phase.

Technical architecture

End-to-end flow: chat and dashboard clients → FastAPI → Gemini-powered agent with tool dispatch → SQLAlchemy / SQLite.

Customer Chat / Admin Dashboard | FastAPI | Gemini AI Agent + Tool Dispatcher | SQLAlchemy / SQLite | Orders, Products, Shipments, Tasks, Messages

When the Gemini API is unavailable, a deterministic fallback calls the same tool functions, so the demo and test suite keep working without an API key.

AI approach

  • Intent classes: order status, product info, shipment status, stock alert, daily briefing, general.
  • Example tools: get_order_status, get_product_info, get_cargo_status, check_stock_alerts, draft_supplier_email, generate_daily_briefing, send_manager_alert.

The app's primary demo language is Turkish — the AI understands and replies in Turkish, and Turkish status strings (e.g. "Gecikmiş", "Kargoda") stay consistent with the business logic and test suite.
